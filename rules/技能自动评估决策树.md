# 技能自动评估决策树 (Skill Decision Tree)

> **用途**: 作为独立规则文件供其他规则用 `@` 引用，决定技能自动路由。
> **前提**: 先读取 `@rules/stack.config.json` 确认激活的模块。

```
START
│
├─ 读取 @rules/stack.config.json 确认激活模块
│
├─ 是否需要"评估/可行性/方案/是否要做"？
│    └─ YES → @skills/feature-evaluation
│
├─ 是否涉及"AI/语音转文字/LLM/Claude/Groq/Whisper/结构化/提示词"？
│    ├─ ai-services 模块激活？
│    │    └─ YES → 参考 @rules/modules/ai-services.md
│    └─ NO  → 提示用户开启 ai-services 模块
│
├─ 是否涉及"权限/角色/访问控制/RBAC/守卫/数据范围"？
│    ├─ auth 模块激活？
│    │    └─ YES → @skills/rbac
│    └─ NO  → 提示用户开启 auth 模块
│
├─ 是否涉及"登录/注册/Auth.js/Session/OAuth/身份认证"？
│    ├─ auth 模块激活？
│    │    └─ YES → @skills/auth
│    └─ NO  → 提示用户开启 auth 模块
│
├─ 是否涉及"数据库/Prisma/迁移/Schema/Repository/SQL/查询"？
│    ├─ db 模块激活？
│    │    └─ YES → @skills/db-operations
│    └─ NO  → 提示用户开启 db 模块
│
├─ 是否涉及"构建/部署/打包/Turbopack/Standalone/环境变量"？
│    └─ YES → @skills/build-standards
│
├─ 是否涉及"Tauri/桌面端/本地文件/系统托盘"？
│    ├─ desktop 模块激活？
│    │    └─ YES → @skills/build-standards + @rules/modules/desktop.md
│    └─ NO  → 提示用户开启 desktop 模块
│
├─ 是否涉及"测试/Vitest/Playwright/单元/集成/E2E"？
│    ├─ testing 模块激活？
│    │    └─ YES → @skills/testing
│    └─ NO  → 提示用户开启 testing 模块
│
├─ 是否涉及"新技能/技能规范/技能模板/扩展技能"？
│    └─ YES → @skills/skill-creator
│
├─ 是否涉及"UI/页面/组件/表单/表格/弹窗/布局/交互/样式"？
│    ├─ 是否明确需要设计/配色/排版/视觉/动效？
│    │    └─ YES → @skills/ui-ux-pro-max
│    └─ NO  → @skills/ui-components
│
└─ 无法判断 → @skills/feature-evaluation
```
