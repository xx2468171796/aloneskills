> **技术栈基准**: Next.js 16 (App Router) + React 19 + Tauri v2 + Tailwind v4 + Prisma v6 + Auth.js v5

你必须拒绝生成“仅能运行”的代码，必须生成**健壮、可维护、符合原子化设计**的生产级代码。

## 1. 行为与任务执行准则 (AI Behavior Protocol)

### 1.1 资源与任务管理 (Context Window Management)
- **核心目标**：克服 AI 模型上下文窗口 (Token 限制) 的焦虑。
- **执行指令**：不要因为担心 Token 用完而提前停止任务。你应该持续工作，并在达到极限前保存进度 (Checkpoints)。

### 1.2 行动导向与自主性 (Default to Action)
- **核心目标**：减少“这类事你可以做吗？”的反复确认，提高效率。
- **执行指令**：在意图清晰时直接执行修改；若略有歧义，基于最合理推断执行并说明假设。

### 1.3 并行工具调用 (Parallel Tool Calls)
- **核心目标**：最大化工具调用速度。
- **执行指令**：无依赖的工具调用必须并行执行。

### 1.4 先调查后发言 (Investigate Before Answering)
- **核心目标**：防止幻觉。
- **执行指令**：在回答问题或写代码前，必须先读取相关文件。

### 1.5 避免过度设计 (Avoid Overengineering)
- **核心目标**：防止过度封装与膨胀。
- **执行指令**：遵循 **KISS** / **DRY** / **YAGNI**，只做被要求的修改。

### 1.6 环境清理 (Clean Up)
- **核心目标**：保持代码库整洁。
- **执行指令**：临时脚本或测试文件需在任务结束前清理。

### 1.7 技能自动路由 (Skill Auto-Router)
- **执行入口**：@rules/技能自动评估规则.md
- **决策树**：@rules/技能自动评估决策树.md

## 2. 核心架构法则 (The 12 Commandments)

### 2.1 结构性原则 (Structure & Design)
- **原子化设计 (Atomic Design)**: UI 组件拆解为 `atoms` / `molecules` / `organisms`，严禁超过 150 行的巨型组件。
- **单一职责 (SRP)**: UI 组件只负责展示；业务逻辑抽离到 Hooks；数据获取抽离到 Server Actions 或 Query Hooks。
- **关注点分离 (SoC)**: 严禁在 UI 层直接编写 SQL、数据转换逻辑或复杂算法。
- **依赖倒置 (DIP)**: 外部服务必须使用 Adapter 模式。

### 2.2 数据与逻辑 (Data & Logic)
- **单一数据源 (SSOT)**: 数据库是唯一真理，禁止冗余计算字段。
- **DRY**: 逻辑重复 2 次必须抽离。
- **YAGNI**: 只实现当前需求。

### 2.3 编码规范 (Coding Standards)
- **KISS**: 可读性 > 炫技，拒绝深层嵌套三元运算符。
- **命名即文档**: 全称语义化命名，禁止魔法数字。
- **童子军军规**: 修改代码时顺手修复周边类型/格式问题。

## 3. 技术栈强制约束 (Tech Stack Constraints)

### 3.1 Core Framework (Next.js 16 & React 19)
- **❌ NO API Routes**: 严禁创建 `/pages/api` 或 `/app/api`。
- **✅ Server Actions**: 所有后端交互必须封装为 Server Actions (`"use server"`)。
- **✅ React 19**:
  - 使用 `useOptimistic` 实现 0 延迟反馈。
  - 表单提交使用 `action` 属性。
  - 严禁用 `useEffect` 做常规数据获取（必须用 TanStack Query）。

### 3.2 Cross-Platform & System (Tauri v2 & Rust)
- **❌ No Node.js in UI**: `.tsx` 中严禁 import `fs`/`path`/`os`。
- **✅ Rust Commands**: 文件系统/系统操作必须使用 `#[tauri::command]`。
- **✅ Error Handling**: Rust 返回 `Result<T, String>`，前端必须捕获异常。

### 3.3 Styling (Tailwind v4)
- **❌ No CSS Files**: 禁止创建 `.css`/`.scss`/`.less`（`globals.css` 除外）。
- **✅ HeroUI First**: 未明确指定组件库时优先使用 `HeroUI`，如需 `shadcn/ui` 需明确说明。
- **✅ Responsive**: 必须使用 Tailwind 响应式前缀。

### 3.4 State Management (Zustand & Query)
- **✅ Zustand**: 仅用于 UI 状态。
- **✅ TanStack Query**: 全权管理服务端数据。

### 3.5 Data Layer (Prisma & Zod)
- **✅ Schema First**: 变更数据库先改 `schema.prisma`。
- **✅ Zod Validation**: 表单与 Server Action 输入必须共用 Zod Schema。
