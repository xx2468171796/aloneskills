# 技能自动评估规则 (Windsurf Skill Auto-Router)

> **用途**: 让 Windsurf 在接到任务时自动判断需要激活的技能，确保指令链路一致与高质量输出。

---

## 1. 路由总流程 (OpenSpec-Style Routing Flow)

> **目标**：像 OpenSpec 一样“先校验 → 再决策 → 再执行”，确保技能调用可追踪、可复现。

### 1.1 前置检查 (Context Gate)

在匹配技能前，必须完成：

1. **读取规则入口**：@rules/通用规则.md
2. **读取决策树**：@rules/技能自动评估决策树.md
3. **确认请求类型**：变更/实现/评估/修复/规范/模板

### 1.2 决策流程 (Deterministic Routing)

1. **解析用户意图**：判断是 UI / 构建 / 认证 / DB / 权限 / 测试 / 功能评估 / 技能创建等。
2. **匹配技能关键词**：按下方“技能触发条件”匹配，命中即激活对应技能。
3. **冲突处理**：如果同时命中多个技能，按“优先级规则”排序激活。
4. **不确定处理**：若无法判断，先调用 `feature-evaluation` 做评估，再进入对应技能。

### 1.3 结果校验 (Routing Validation)

- **必须输出**：命中的触发关键词与对应技能
- **必须解释**：优先级排序与被抑制的技能
- **禁止跳过**：任何未满足触发条件的技能不可调用

## 3. 技能触发条件 (Skill Triggers)

### 3.1 UI / UX
- **触发关键词**：页面/组件/表单/弹窗/表格/交互/布局/配色/字体/视觉/UX
- **激活技能**：
  - `ui-components`（组件实现）
  - `ui-ux-pro-max`（设计/视觉/交互强化）

### 3.2 构建与环境
- **触发关键词**：构建/部署/打包/环境变量/standalone/Turbopack/Tauri
- **激活技能**：`build-standards`

### 3.3 数据库与数据流
- **触发关键词**：Prisma/SQL/数据库/迁移/查询/Repository/Schema
- **激活技能**：`db-operations`

### 3.4 认证与会话
- **触发关键词**：登录/注册/Auth.js/Session/OAuth/权限登录
- **激活技能**：`auth`

### 3.5 权限与角色
- **触发关键词**：RBAC/角色/权限/守卫/数据范围/权限控制
- **激活技能**：`rbac`

### 3.6 测试
- **触发关键词**：测试/单元测试/集成测试/E2E/Vitest/Playwright
- **激活技能**：`testing`

### 3.7 新功能评估
- **触发关键词**：新功能/方案评估/可行性/是否需要/架构评审
- **激活技能**：`feature-evaluation`

### 3.8 创建或扩展技能
- **触发关键词**：新技能/技能规范/技能模板/更新技能
- **激活技能**：`skill-creator`

---

## 4. 优先级规则 (Priority)

1. **feature-evaluation**：需求不明确或涉及新能力时，优先评估
2. **rbac/auth**：涉及安全/权限必须优先于 UI 或数据实现
3. **db-operations**：涉及数据结构与变更优先
4. **build-standards**：构建/部署/环境优先于 UI 实现
5. **ui-components/ui-ux-pro-max**：实现与视觉优化
6. **testing**：收尾或补充阶段
7. **skill-creator**：仅在显式创建/扩展技能时启用

---

## 5. 规则落地示例 (Routing Examples)

- **“做一个用户列表页+筛选”** → `ui-components`（必要时叠加 `db-operations`）
- **“我要加权限控制”** → `rbac`（优先）
- **“要评估是否需要引入新模块”** → `feature-evaluation`
- **“构建失败/部署配置”** → `build-standards`

---

## 6. 执行要求 (Execution Requirements)

- **必须先读文件再改动**（防幻觉）
- **不满足触发条件时不得强行激活技能**
- **多技能并发时必须解释选择顺序**
- **执行前复核**：若触发条件/目标不明确，先调用 `feature-evaluation`
