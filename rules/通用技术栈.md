# Project Core: The Ultimate Technical Foundation (v2.1)

> **文档版本**: 2.1.0 (Standalone Infra Edition)
> **文档日期**: 2026-01-26
> **部署策略**: **应用容器化 / 基建独立化**。
> **核心目标**: 构建一个基于 **Next.js 16** 和 **Tauri v2** 的高性能 Monorepo 基座，连接外部独立部署的基础设施。

---

## 1. 核心引擎 (The Core Engines)

**架构哲学**: Web First, Desktop Native, Build Fast.

| 模块 | 技术选型 | 版本 | 核心说明                                         |
| :----- | :--------- | :----- | :------------------------------------------------- |
| **Web 核心**     | **Next.js**         | **v16**     | App Router 架构。**强制开启 Turbopack** (`--turbo`)。                           |
| **桌面核心**     | **Tauri**         | **v2.0**     | **Windows 原生能力** (Rust + WebView2)。负责本地文件 IO、系统托盘。  |
| **构建系统**     | **Turborepo**         | **Latest**     | 智能增量构建。管理 Monorepo 任务依赖与远程缓存。 |
| **包管理器**     | **Yarn**         | **v4 (Berry)**     | 启用 Workspaces 模式。严格依赖管理。             |
| **语言标准**     | **TypeScript**         | **v5.x**     | **Strict Mode** (全量严格模式)。全链路类型安全。                |

---

## 2. UI 工程化 (UI Engineering)

**架构哲学**: 原子化设计，组件驱动开发。

| 模块 | 技术选型 | 版本 | 核心说明                      |
| :----- | :--------- | :----- | :------------------------------ |
| **组件库**     | **HeroUI**         | **Latest**     | 默认组件库；需要 `shadcn/ui` 时需明确指定。   |
| **样式引擎**     | **Tailwind CSS**         | **v4.0**     | **Oxide Engine**。无 PostCSS 配置，编译极快。 |
| **组件调试**     | **Storybook**         | **v8**     | 独立组件开发环境，运行于 `packages/ui`。   |
| **状态管理**     | **Zustand**         | v5   | 客户端 UI 状态管理。          |
| **数据同步**     | **TanStack Query**         | v5   | 服务端数据状态同步与缓存。    |

---

## 3. 后端与中台基座 (Backend Foundation)

**架构哲学**: Stateless Compute (无状态计算) + External State (外部状态)。

| 模块 | 技术选型 | 核心说明                                |
| :----- | :--------- | :---------------------------------------- |
| **API 范式**     | **Server Actions**         | React 19 Native。直接连接外部数据库。   |
| **管理中台**     | **Payload CMS**         | **v3.0 (Native)** 。嵌入 Next.js 运行。**配置外部 PG 和 S3 连接**。                 |
| **ORM**     | **Prisma**         | **v6+** 。启用 `multiSchema`。用于管理外部 PostgreSQL 结构。 |
| **身份认证**     | **Auth.js**         | **v5**。Session 存储于外部 Redis。            |

---

## 4. 外部基础设施接入 (External Infrastructure)

**重要**：本项目不包含基础设施的 Docker 编排。所有服务通过 **环境变量** 连接到独立部署的实例。

| 服务 | 协议/版本要求 | 环境变量示例 (.env) |
| :----- | :-------------- | :-------------------- |
| **数据库**     | **PostgreSQL v17+**              | `DATABASE_URL="postgresql://user:pass@192.168.x.x:5432/nexus?schema=public"`                    |
| **缓存**     | **Redis v7+**              | `REDIS_URL="redis://:password@192.168.x.x:6379/0"`                    |
| **存储**     | **S3 Compatible**              | `S3_ENDPOINT="https://minio.internal.com"`<br />`S3_ACCESS_KEY="..."`<br />`S3_SECRET_KEY="..."`                |

---

## 5. 目录结构规范 (Directory Structure)

```text
/
├── .cursor/              # Windsurf Rules
├── .env.example          # [关键] 环境变量模版 (配置外部连接)
├── package.json          # Yarn Workspaces Root
├── turbo.json            # Turborepo Config
├── biome.json            # Biome Linter Config
│
├── apps/
│   └── portal/           # [核心应用] Next.js 16 + Payload v3 + Tauri v2
│       ├── src/          # React 源码
│       ├── src-tauri/    # Rust 源码
│       ├── payload/      # Payload CMS Config
│       └── .env          # [应用级配置] 必须包含 DB/Redis/S3 连接串
│
└── packages/             # [共享基座]
    ├── ui/               # [UI Core] HeroUI + Tailwind + Storybook
    ├── db/               # [Data Core] Prisma Client (连接外部 DB)
    ├── config/           # [Config Core] Shared TSConfig
    └── utils/            # [Logic Core] Shared Utils
```
