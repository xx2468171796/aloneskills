# 工作流模式

## 顺序工作流

```
Step 1: 执行 A
Step 2: 执行 B
Step 3: 执行 C
```

适用于：线性任务，每步依赖上一步的输出

## 条件工作流

```
Step 1: 检查条件 X
  - 如果 X 为真 → 执行 A
  - 如果 X 为假 → 执行 B
Step 2: 继续...
```

适用于：有分支逻辑的任务

## 决策树工作流

```
收到请求
├─ 情况 A？ → 处理 A
├─ 情况 B？ → 处理 B
├─ 情况 C？ → 处理 C
└─ 默认 → 处理 D
```

适用于：多分支决策

## 循环工作流

```
Step 1: 初始化列表
Step 2: 对每个项目：
  - 执行操作
  - 记录结果
Step 3: 汇总结果
```

适用于：批量处理任务

## 验证工作流

```
Step 1: 接收输入
Step 2: 验证格式
  - 失败 → 返回错误
Step 3: 验证业务规则
  - 失败 → 返回错误
Step 4: 处理输入
Step 5: 返回结果
```

适用于：需要多层验证的任务
