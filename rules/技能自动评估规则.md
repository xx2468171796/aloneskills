# 技能自动评估规则 (Skill Auto-Router)

> **用途**: AI 接到任务时自动判断需要激活的技能，确保指令链路一致与高质量输出。

---

## 1. 路由总流程

### 1.1 前置检查 (Context Gate)

在匹配技能前，必须完成：

1. **读取模块配置**：`@rules/stack.config.json` — 确认当前项目激活了哪些模块
2. **读取核心规则**：`@rules/core.md` + `@rules/coding-standards.md`
3. **读取决策树**：`@rules/技能自动评估决策树.md`
4. **确认请求类型**：变更/实现/评估/修复/规范/模板

### 1.2 模块感知路由 (Module-Aware Routing)

**关键规则**：只激活当前项目已启用模块对应的技能。

1. **解析用户意图**：判断是 UI / 构建 / 认证 / DB / 权限 / 测试 / AI 等
2. **检查模块开关**：该任务对应的模块在 `stack.config.json` 中是否为 `true`
3. **匹配技能关键词**：按"技能触发条件"匹配，命中即激活
4. **冲突处理**：同时命中多个技能，按"优先级规则"排序
5. **模块未激活处理**：如果任务需要未激活的模块，**先提示用户是否需要开启该模块**

### 1.3 结果校验

- **必须输出**：命中的触发关键词与对应技能
- **必须说明**：是否受模块开关影响
- **禁止跳过**：未满足触发条件的技能不可调用

## 3. 技能触发条件

### 3.1 UI / UX
- **触发关键词**：页面/组件/表单/弹窗/表格/交互/布局/配色/字体/视觉/UX
- **激活技能**：`ui-components` / `ui-ux-pro-max`
- **所需模块**：core（必选，始终激活）

### 3.2 构建与环境
- **触发关键词**：构建/部署/打包/环境变量/standalone/Turbopack/Tauri
- **激活技能**：`build-standards`
- **所需模块**：core；涉及 Tauri 时需 `desktop` 模块

### 3.3 数据库与数据流
- **触发关键词**：Prisma/SQL/数据库/迁移/查询/Repository/Schema
- **激活技能**：`db-operations`
- **所需模块**：`db`

### 3.4 认证与会话
- **触发关键词**：登录/注册/Auth.js/Session/OAuth/权限登录
- **激活技能**：`auth`
- **所需模块**：`auth`

### 3.5 权限与角色
- **触发关键词**：RBAC/角色/权限/守卫/数据范围/权限控制
- **激活技能**：`rbac`
- **所需模块**：`auth`

### 3.6 测试
- **触发关键词**：测试/单元测试/集成测试/E2E/Vitest/Playwright
- **激活技能**：`testing`
- **所需模块**：`testing`

### 3.7 AI 服务
- **触发关键词**：AI/语音转文字/LLM/Claude/Groq/Whisper/转写/结构化/提示词
- **激活技能**：参考 `@rules/modules/ai-services.md`
- **所需模块**：`ai-services`

### 3.8 新功能评估
- **触发关键词**：新功能/方案评估/可行性/是否需要/架构评审
- **激活技能**：`feature-evaluation`
- **所需模块**：core（始终可用）

### 3.9 创建或扩展技能
- **触发关键词**：新技能/技能规范/技能模板/更新技能
- **激活技能**：`skill-creator`
- **所需模块**：core（始终可用）

---

## 4. 优先级规则

1. **feature-evaluation** — 需求不明确或涉及新能力时，优先评估
2. **rbac / auth** — 涉及安全/权限必须优先于 UI 或数据实现
3. **db-operations** — 涉及数据结构与变更优先
4. **ai-services** — 涉及 AI 集成
5. **build-standards** — 构建/部署/环境优先于 UI 实现
6. **ui-components / ui-ux-pro-max** — 实现与视觉优化
7. **testing** — 收尾或补充阶段
8. **skill-creator** — 仅在显式创建/扩展技能时启用

---

## 5. 执行要求

- **必须先读文件再改动**（防幻觉）
- **不满足触发条件时不得强行激活技能**
- **多技能并发时必须解释选择顺序**
- **模块未激活时先提示用户，不要静默跳过**
- **版本号引用 `@rules/versions.md`，不要在代码中写死版本**
